Why is this an issue?

The rule "Explicit return types MUST BE declared on functions" is a coding standard guideline often used in modern PHP development, especially with strict typing enabled.

It means that every function or method you define should specify the type of value that it will return.

This helps with code clarity, maintenance, and avoids potential runtime errors due to mismatched types.

function doSomething() :string // compilant
{
    return 'test';
}

function doSomething(){ // Noncompliant
    return 'test';
}

Exceptions

Magic methods (__construct(), __destruct(), __call(), __callStatic(), __get(), __set(), __isset(), __unset(), __sleep(), __wakeup(), __toString(), __invoke(), __set_state(), __clone(), __debugInfo()) are ignored.

function __construct(){...} // Compliant by exception
function __destruct(){...}  // Compliant by exception
