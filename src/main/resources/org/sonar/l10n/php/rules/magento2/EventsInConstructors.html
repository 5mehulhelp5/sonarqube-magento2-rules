Why is this an issue?

In Magento 2, triggering events within constructors is considered a bad practice. When events are dispatched during object construction, it can lead to unpredictable behavior, potential dependency cycles and Performance degradation during object initialization 

    public function __construct($fileReader, $eventManager)
    {
        $this->data = $fileReader->read('cache.xml');
        $eventManager->dispatch('config_read_after'); // Noncompliant
    }
