Why is this an issue?

The rule aims to detect legacy practices for block classes and their methods

Noncompliant Code Example

$block->getChildHtml('block_name', true, true); // Noncompliant {{3rd parameter is not needed anymore for getChildHtml()}}
$block->getChildChildHtml('a', 'b', 'c', 'd'); // Noncompliant {{4th parameter is not needed anymore for getChildChildHtml()}}

Compliant Solution

$block->getChildHtml('block_name', true); //Compliant
$block->getChildChildHtml('a', 'b', 'c'); //Compliant

Methods to check

  • getChildHtml
  • getChildChildHtml
